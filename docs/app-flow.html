Verification engine — backend/app/
1

Vision extraction claude_client.py

One Claude call returns structured JSON: every field as printed, a verbatim transcription of the government warning, formatting attributes (header_all_caps, header_bold), and a legibility grade. No correcting, no guessing.

the model transcribes — it never decides
2

Deterministic checks matching.py

Code compares the transcription against the statutory warning word-for-word (27 CFR 16.21) and verifies the ALL-CAPS header; parses ABV numerically and cross-checks proof = 2×ABV; converts volume units (750 mL = 75 cl); applies a fuzzy band so Stone's Throw matches STONE'S THROW with a note.

the code decides — every rule unit-tested
3

Verdict rollup matching.py

Per-field results roll up conservatively: any deterministic violation rejects; anything unconfirmable — fuzzy borderline, poor legibility, bold that can't be verified from the photo — routes to a human instead of silently passing.

triage, not adjudication
Lane A · response

JSON result card

Verdict plus a per-field table: application value vs. label value, with the reason for every flag — the agent sees why, then accepts or rejects.

~2–4 s · within the 5 s budget
Lane B · response
ADR-007

NDJSON stream, completion order

One result per line, the moment it finishes. Rows render live with an n / total progress count — minutes of visible progress, never a spinner. A failed label streams an ERROR row; the rest of the batch is untouched.
